# Decisions (ADRs)

Architecture Decision Records capture significant choices: the context, the decision, and its consequences. They are short, numbered, and append-only — when a decision changes, add a new ADR that supersedes the old one rather than rewriting history.

## Format

Each ADR: **Context** (the situation/problem) → **Decision** (what was chosen) → **Consequences** (trade-offs, follow-ups), plus a **Status** (proposed / accepted / superseded).
